Overview

Physically unclonable functions (PUFs) return a unique response to a challenge, expose manufacturing variance intrinsic to IC fabrication, and currently require dedicated hardware and proprietary software.

The PUFLib project aims to further the state-of-the-art of PUF research, and provide a simple cross-platform API to utilize, manipulate, and form a trusted sealed storage mechanism based on these constructs.

Our goal with PUFLib is to develop an open-source platform for contributions: a modular library to use PUFs for data protection using Intel x86 and ARM, a variety of “legacy” and novel PUF plug-ins, and a modular design to facilitate contributions. We also plan to provide seal()/unseal() semantics for device-specific sealed storage.

Our approach allows us to incorporate sources such as:

  • SRAM
  • Flash
  • FPGA
  • SMBus EEPROM

Roadmap

End of 2016: Preliminary support for EEPROM on some memory modules

Middle of 2017: Preliminary support for DDR PUF source, and web cam evaluation for PUF characterization

End of 2017: Beta support for EEPROM, DDR, and CMOS sensor

Compilation

Documentation can be generated with make docs ; make all will build PUFLib for the system.

The PUFLib core includes automatic provisioning of hardware-supported PUF sources, error-correcting codes for temp/age variance, and Shamir’s secret sharing for multi-PUF support and additional error tolerance.
